# Messaging Feature

## What We Built

We built in-app messaging between patient and psychologist.

## Main Actions

- Start conversation
- Send message
- Read conversation history
- Show unread count

## Routes and APIs

- Pages: `/patient/dashboard/messages`, `/psychologist/dashboard/messages`
- APIs: `/api/messages/start`, `/api/messages/conversations`, `/api/messages/[appointmentId]`, `/api/messages/unread`

## Notes

We now use "Message Now" behavior from find-psychologists flow.
